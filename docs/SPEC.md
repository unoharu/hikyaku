# 飛脚（Hikyaku）開発仕様書

## 1. プロジェクト概要

`hikyaku` は Go 製の「遊び心」特化型ファイル操作 CLI ツール。
`cp` / `mv` の機能を、江戸時代の飛脚が荷物を運ぶ体験として昇華させる。

### 設計思想

- **不便を楽しむ:** 転送速度より転送の「ドラマ」を重視する
- **粋なUX:** エラーは叱咤、成功は祝い、として江戸っ子口調で表現する
- **Goの親和性:** シングルバイナリで動作する軽量な CLI

---

## 2. コマンド体系

| コマンド                     | 引数     | 説明                                                         |
| :--------------------------- | :------- | :----------------------------------------------------------- |
| `hikyaku run [src] [dst]`    | src, dst | **荷運び（コピー）**: 飛脚が走りながらファイルをコピーする   |
| `hikyaku todoke [src] [dst]` | src, dst | **届け（移動）**: ファイルを移動し、元場所を「空き地」にする |
| `hikyaku kawaraban [path]`   | path     | **瓦版（一覧）**: ディレクトリを木札風レイアウトで表示する   |
| `hikyaku meibo`              | -        | **台帳（履歴）**: 過去の転送記録と飛脚の称号を表示する       |

---

## 3. 優先度分類

### Must（Phase 1〜2 で必ず実装）

- `run` / `todoke` コマンドの基本動作（ファイルコピー・移動）
- 江戸っ子口調メッセージ（成功・確認・中断・エラー）
- 飛脚プログレスバー（BubbleTea 使用）
- 江戸単位変換による転送量表示

### Should（Phase 2〜3 で実装）

- `kawaraban` コマンド（木札風ディレクトリ一覧）
- `--yonige` フラグ（バックグラウンド実行時の静音モード）
- `--kakugo` フラグ（上書き確認スキップの江戸版）
- 荷物の重さコメント（ファイルサイズに応じた台詞）
- 天気ランダムイベント（実行ごとに晴れ/雨/嵐）
- 飛脚の独り言（転送中にランダムで挿入）
- 空ファイルイベント（サイズ0ファイルへの反応）

### Could（Phase 3 以降・余力があれば）

- 宿場町ルート表示（東海道53次を進捗バーに組み込む）
- 飛脚ランク制度（累積転送量で称号が変わる）
- `meibo` コマンド（転送台帳・履歴表示）
- おみくじ機能（ファイルのハッシュ値で運勢を表示）
- 盗賊イベント（1% の確率で転送が 5 秒止まる）
- 時そばタイマー（進捗を「今、何時だい？」で問う）

## 4. UI / UX 設計

### 4.1 飛脚プログレスバー

```text
出発地点：[./old_data] ---------------------------------- 到着地点：[/new_disk]
宿場町[日本橋]　　　　　　　　　　　　　　　　　　　　　　　宿場町[三条大橋]
[━━━━━━━━━━━━━━━🏃💨　　　　　　　　　　　　　] 45%
　　　　　　「へい、今は箱根の山越えってところだ。きついぜ！」
```

- BubbleTea の `progress` コンポーネントで実装
- 飛脚キャラクター（🏃💨）がバーに沿って移動するアニメーション

#### 宿場町ルート（Could）

東海道53次の宿場を配列で定義し、進捗%に応じて「現在地」を表示する。

```text
0%   → 日本橋
15%  → 箱根
50%  → 浜松
85%  → 草津
100% → 三条大橋
```

#### 天気ランダムイベント（Should）

実行開始時にランダムで天気を決定し、プログレスバーの台詞に反映する。

| 天気 | 台詞                           |
| :--- | :----------------------------- |
| 晴れ | 「追い風だ！気分が上がるぜ！」 |
| 雨   | 「足元がぬかるんでいやがる…」  |
| 嵐   | 「台風だー！荷が濡れちまう！」 |

#### 飛脚の独り言（Should）

転送中、数十秒ごとにランダムで1行挿入する。

- 「腹が減ったな…もう少しで茶屋があるはずだ」
- 「このあたりで昔、狐に化かされたって話があるぜ」
- 「雨が降ってきやがった。でも飛脚は止まらねぇ！」

## 5. 飛脚ランク制度（Could）

累積転送量を `~/.hikyaku/state.json` に記録し、称号を決定する。

| 累積転送量 | 称号         |
| :--------- | :----------- |
| 〜1GB      | 見習い飛脚   |
| 〜10GB     | 一人前の飛脚 |
| 〜100GB    | 名飛脚       |
| 100GB〜    | 伝説の飛脚   |

`meibo` コマンドと起動時ヘルプで称号を表示する。

## 6. meibo コマンド仕様（Could）

転送ログを `~/.hikyaku/log.json` に追記し、`meibo` で一覧表示する。

```text

────────────────────────────────────────────
天保元年三月一日  ./old → /new   三尺二分  大吉
天保元年三月二日  ./src → /dst   一寸五厘  凶
────────────────────────────────────────────
累計：四尺七分七厘
```

## 7. 隠し機能の仕様（Phase 3 以降）

### おみくじ

- コピー完了時、ファイルの SHA-256 ハッシュの末尾バイトで運勢を決定
- 例：「大吉：このコードはバグが出ねぇ予感だぜ！」

### 盗賊イベント

- `run` / `todoke` 実行時に 1% の確率で発生
- 「泥棒だー！」と表示し、転送を 5 秒停止
- 5 秒後「取り返したぜ！」で再開

### --yonige フラグ（静音モード）

- バックグラウンド実行時に使用
- メッセージが「...シッ、静かにな」に変わり、プログレスバーを非表示にする

### 時そばタイマー

- 待機時間が長い際、「今、何時だい？」と問うと「へい、九つ（90%）で！」と返す
- インタラクティブな入力インターフェースが必要
